# 疯狂奶茶杯

基于 Pygame 的脑机接口（BCI）奶茶制作游戏。玩家通过键盘或头环控制奶茶杯，接住从天而降的食材，完成一杯杯奶茶。通过科创平台实时获取专注力和头动数据，专注力影响食材速度和收益系数，最终累积营业额提升等级。

## 主菜单与控制模式

主菜单包含 4 个按钮（2×2 布局）：游戏开始、选择模式、游戏设置、退出。

### 控制模式

| 模式               | 说明                                                                           | 底层游戏配置                                                |
| ------------------ | ------------------------------------------------------------------------------ | ----------------------------------------------------------- |
| **实验模式** | 3 段式自动轮转：3min 热身原萃 + 7min 特调 + 5min 忆调。热身阶段收集专注力基线 | 热身阶段使用原萃底层（has_required、粒子效果、杯结算），后续阶段切换对应底层 |
| **特调模式** | BCI 头环控制杯子移动。点击开始后尝试连接 BCI，连接失败弹窗提示后降级为键盘控制 | 使用 BCI 游戏底层（45杯、自由搭配），第一杯后计算归一化范围 |
| **忆调模式** | 记忆配方食材序列，然后按序接住。专注力调节食材速度，15 分钟训练 | 独立游戏逻辑，预生成投放序列，食材落地后结束轮次 |
| **原萃模式** | 原始注意力值直接驱动。专注力不经归一化，直接控制食材速度和收益系数             | 基于特调模式底层（45杯），跳过归一化环节                    |

### BCI 未连接时的行为

- 特调模式 / 原萃模式 BCI 失败 → `control_mode = "bci_failed"` → 使用键盘控制，直接进入正式游戏
- 对局**不保留历史记录**，得分**不计入等级累计**
- 音效正常工作（接食材、必接、结算、升级等）

## 车道系统

### 六车道配置

画面分为 **6 条垂直车道**，每条宽约 213px（1280÷6）：

```
┌──────┬──────┬─────────────┬──────┬──────┐
│ 车道0 │ 车道1 │  车道2+3合并  │ 车道4 │ 车道5 │
│ 掉落  │ 掉落  │  无分割线    │ 掉落  │ 掉落  │
│ 小料  │ 小料  │  不掉落食材  │ 小料  │ 小料  │
└──────┴──────┴─────────────┴──────┴──────┘
```

- **中间两条车道（车道2、3）**：合并为一个宽区域，无分割线，**不掉落食材**
- **左右四条车道（车道0、1、4、5）**：随机掉落食材
- 每条车道最多同时有 1 个食材（由 `_free_lanes()` 占用检测保证）

### 车道占用检测算法

仅检查屏幕上方 35% 区域的食材，底部即将消失的食材不阻塞新生成：

```
occupied = set()
for ing in ingredients:
    if ing.rect.y < SCREEN_HEIGHT * 0.35:     # 仅上方区域
        lane = min(ing.centerx // LANE_WIDTH, 5)  # clamp 防越界
        occupied.add(lane)
free_lanes = [l for l in [0, 1, 4, 5] if l not in occupied]
```

### 食材生成位置

随机从空闲车道中选取一条，在该车道宽度范围内随机生成 X 坐标：

```
lane = random.choice(free_lanes)
x = random.randint(lane * LANE_WIDTH, (lane + 1) * LANE_WIDTH - INGREDIENT_SIZE)
```

## 实验模式

实验模式是 **3 段式自动轮转** 的新型训练模式：

```
热身阶段 (3min)  →  特调阶段 (7min)  →  忆调阶段 (5min)
   原萃驱动           正式游戏             记忆训练
```

### 热身阶段

进入实验模式后，先弹出 2.5 秒热身提示，**画面和计时冻结**。弹窗消失后正式开始 3 分钟热身。

- **玩法**：原萃模式逻辑 — 原始专注力值直接驱动食材速度
- **必接食材**：基于当前等级 INGREDIENT\_TIERS，必接食材带粒子效果
- **杯结算**：每杯必须接住必接食材才算成功，杯数 +1；失败则不计数
- **秘方翻倍**：专注力持续高于杯基线+10 达 8 秒触发秘方弹窗
- **生成间隔**：动态调节（speed\_ratio → 调整后间隔），防止食材过密
- **右下角**：显示 "热身阶段" 标签 + "热身 M:SS" 倒计时

**热身结束 — 归一化基准计算**：

取**最后 30 秒**注意力数据，计算个性化归一化范围：

```
基线 = 全场注意力均值
上界 = 最后30秒最高注意力（上限 100）
下界 = 最后30秒最低注意力（下限 0）
若 上界 - 下界 < 5：向两侧各扩展 10，保证范围宽度至少 20
```

结果为后续特调阶段提供归一化参考。

> **注意**：特调阶段和忆调阶段为后续迭代内容，热身结束后自动返回主菜单。

## 正式游戏机制

### 一杯制规则

- 一局最大 **15 分钟（900 秒）**，共 **45 杯**，每杯最长 **20 秒**（`CUP_DURATION = 20`）
- **杯结束结算逻辑**（`CupManager.settle_cup()`）：

  1. 无必接食材 → 本杯金额 = 0
  2. 有必接食材 → 本杯金额 = 所有接住食材分值之和
  3. 触发秘方 → 本杯金额 × 2（触发即翻倍，弹窗提示后自动生效）

### 食材分值表

| 食材       | 分值 | 食材     | 分值 | 食材   | 分值 |
| ---------- | ---- | -------- | ---- | ------ | ---- |
| 珍珠       | 1    | 椰果     | 1    | 牛奶   | 2    |
| 红茶       | 2    | 绿茶     | 2    | 芋圆   | 2    |
| 脆啵啵     | 2    | 芒果     | 3    | 椰奶   | 3    |
| 草莓       | 3    | 芋泥     | 3    | 燕麦奶 | 4    |
| 咖啡       | 4    | 抹茶奶盖 | 5    | 米酿   | 5    |
| 咸芝士奶盖 | 6    | 茉莉花茶 | 6    | 冰块   | 0    |

### 专注力收益系数（BCI 模式）

防止低专注力下食材速度快反而接到更多、收益更高。高专注力 → 全额收益，低专注力 → 收益打折：

```
系数(norm) = ┌ 1.0                        , norm ≥ 80
             │ 0.5 + 0.5 × (norm-40)/40   , 40 ≤ norm < 80
             └ 0.5                        , norm < 40
```

| 归一化值 | 系数      | 含义             |
| -------- | --------- | ---------------- |
| ≥ 80    | 1.0       | 全额收益         |
| 40 ~ 80  | 0.5 ~ 1.0 | 线性过渡         |
| < 40     | 0.5       | 惩罚（最低保障） |

杯结算时：`最终金额 = 原始金额 × 系数`

### 正式游戏速度调节（归一化线性映射）

将实时注意力映射到食材速度，动态更新归一化范围（初始 `[30, 70]`，每杯根据最新注意力数据更新）：

```
速度 = FORMAL_SPEED_MAX - (归一化值 - 1) / 99 × (FORMAL_SPEED_MAX - FORMAL_SPEED_MIN)
     = 4.5 - (归一化值 - 1) / 99 × 2.5
```

| 归一化值      | 速度                   | 含义 |
| ------------- | ---------------------- | ---- |
| 1（低专注）   | FORMAL_SPEED_MAX (4.5) | 较快 |
| 50            | 3.25                   | 中等 |
| 100（高专注） | FORMAL_SPEED_MIN (2.0) | 较慢 |

速度变化范围 2.0 ~ 4.5（差 2.5），确保变化感知适度。

### 生成间隔动态调节

生成间隔随速度自适应，使食材密度保持一致：

```
速度比 = 当前速度 / 基础速度
调整后间隔 = 基础间隔 × (0.7 + 0.6 × 速度比)
最终间隔 ∈ [0.3, 3.0]
```

### 4 级食材与等级系统

| 等级 | 升级条件     | 可用食材（含前级）              | 必接食材（含前级） |
| ---- | ------------ | ------------------------------- | ------------------ |
| Lv.1 | 默认         | 珍珠、椰果、牛奶、红茶、绿茶    | 牛奶、红茶、绿茶   |
| Lv.2 | 累计 ≥ 150  | +芋圆、脆啵啵                   | +芒果、椰奶        |
| Lv.3 | 累计 ≥ 400  | +草莓、芋泥                     | +燕麦奶、咖啡      |
| Lv.4 | 累计 ≥ 1000 | +特调稀奶油顶、米酿、咸芝士奶盖 | +茉莉花茶          |

每级保留所有前级食材，等级越高可选种类越多。

### 冰块替换机制（注意力方差自适应）

以**上一杯奶茶的平均专注力**为基线（第1杯使用默认值 40），计算实时注意力与基线的偏离方差：

```
offset_i = 实时注意力_i - 上杯基线
方差 = Σ(offset_i - offset̄)² / N     （60帧滑动窗口）
```

**每杯独立基线计算**：杯结束后取该杯所有注意力样本的均值，作为下一杯的基线。

根据方差动态决定冰块替换概率，方差越大 → 冰块越频繁：

| 方差     | 模式     | 冰块概率 | 含义                                   |
| -------- | -------- | -------- | -------------------------------------- |
| < 50     | 简单模式 | 20%      | 注意力稳定，平均 5 个小料中 1 个变冰块 |
| 50 ~ 150 | 中等模式 | 50%      | 正常波动，平均 2 个小料中 1 个变冰块   |
| > 150    | 困难模式 | 80%      | 波动大，大多数小料被替换为冰块         |

**特殊规则**：方差 ≥ 150 且实时注意力 < 20 → 冰块概率 100%（只掉冰块）。

**冰块机制**：冰块**不是额外掉落**，而是在正常小料生成管线中按概率**替换**原定食材。冰块分值 0，不参与必接判定。玩家视角就是"小料变少，冰块变多"。

### 低专注保护（正式游戏）

- 注意力 **≤ 10 持续 5 秒** → 游戏暂停，半透明黑化遮罩，大字提示"请调整身心状态"
- 注意力 **> 15 持续 3 秒** → 恢复游戏
- 遮罩透明度以指数平滑过渡：`α += (target - α) × 0.05`

### 秘方触发机制

**BCI 模式** — 基于持续高专注触发：

```
第1杯阈值 = 默认归一化基线 + 10
第N杯阈值 = 第N-1杯平均专注力 + 10
若 注意力 > threshold 持续 8 秒 → 触发秘方！
```

**非 BCI 模式** — 基于杯序周期性触发：

- 每 `secret_recipe_cup_interval` 杯触发一次（特调模式每杯触发，BCI 模式每 3 杯触发）

**触发方式改为弹窗**（v2）：秘方不再作为掉落食材出现。触发时屏幕中央弹出半透明弹窗，显示"触发秘方！"文字和秘方图案（左右晃动 + 旋转），持续 2 秒。杯结算时**触发即翻倍**（不再需要接住秘方食材）。

### 防伪迹仲裁（BCI 模式）

正常专注时头部有 1-2 度自然微抖，若检测到异常（刻意咬牙、僵直）则触发惩罚：

- **检测条件**：陀螺仪三轴变化 **均 < 0.5°** 且专注力 **> 80**，持续 **2 秒**
- **惩罚**：屏幕提示"请放松面部肌肉"，画面和时间冻结 **5 秒**，自动恢复
- 恢复后重置陀螺仪基准，防止连续触发

### 最后 5 分钟专注力记录

每局游戏结束时，根据专注力采样和时间推算最近 5 分钟（300 秒）的平均专注力：

```
sps = len(focus_samples) / game_duration           # 每秒采样数
n = max(1, int(sps × 300))                          # 最后 300 秒的样本数
last_5min_avg = mean(focus_samples[-n:])             # 尾部样本均值
```

该值存入历史记录，作为趋势曲线的数据点。

## 忆调模式

### 游戏流程

```
[规则弹窗 3.5s] ← 整个游戏只出现一次
     ↓
┌── 循环 ───────────────────────────────────────────────────┐
│ [记忆阶段 2s]    食材图片并排显示 + 配方名                  │
│ [接食材]         4出料口按预生成序列投放，按序接食材        │
│                 专注力实时调节下落速度（2.5~6.5 px/frame） │
│                 所有食材落地后强制结束本轮                  │
│ [结果显示 1.5s]  成功/未完成/接错判定                      │
│ [休息 2s]        显示等级/进度/已训练时间统计              │
└──────────────────────────────────────────────────────────┘
     ↓ 训练满 15 分钟后，当前轮结束即结算
```

### 核心规则

1. **记忆配方**：屏幕中央显示 2~5 种食材图片和配方名称，停留 2 秒
2. **按序接住**：食材消失后，按预生成的投放序列依次掉落，**必须按配方顺序依次接住**
3. **严格顺序**：接错顺序即本轮失败
4. **本轮结束条件**：所有预投放食材已落地（接到或掉落出屏幕），轮次自动结束。未接完所有配方食材 → 失败
5. **15 分钟训练**：一局总训练时长为 15 分钟（可配置），超时后当前轮结束即结算。若超时发生在接食材阶段，等待本轮完成后结束
6. **专注力调速**：BCI 模式下实时读取专注力值，高专注 → 食材慢（最低 2.5 px/frame），低专注 → 食材快（最高 6.5 px/frame）
7. **穿插投放**：每轮预生成 `recipe_length × 3` 个食材，配方食材每隔 3 个投放位出现一次（目标:干扰 = 1:2）


### 难度进阶

| 等级 | 配方食材数 | 晋级/降级条件         |
| ---- | ---------- | --------------------- |
| Lv1  | 2          | 连续成功 3 次 → 升级 |
| Lv2  | 3          | 连续成功 3 次 → 升级 |
| Lv3  | 4          | 连续成功 3 次 → 升级 |
| Lv4  | 5（上限）  | 保持在最高级          |

- 失败 → 连续成功清零
- 累计失败 2 次 → 降一级 + 失败计数清零

### 车道生成算法

忆调模式使用**独立的 5 车道系统**（不同于主游戏的 6 车道），每条车道宽 256px：

```
lane = _free_lane()       # 随机选取空闲车道
center = lane × 256 + 128  # 车道中心
x = random(center-30, center+30)  # 微调 x 坐标
```

车道空闲判断：该车道内食材数 < `_max_per_lane`（当前为 1）且无食材 `y < _lane_spacing`（400px）。

## 脑机接口（BCI）系统

### 架构概览

```
[科创平台] ←→ [TCP Socket 127.0.0.1:8000] ←→ [BCIDataReader] ←→ [GameSession]
                              │
                   IPC 二进制协议
               (4字节大端长度 + JSON负载)
```

### 连接流程

1. 主菜单选择特调模式并点"游戏开始"
2. 尝试 TCP 连接科创平台（默认 `127.0.0.1:8000`）
3. 成功 → 发送就绪消息 → 启动陀螺仪焦点算法 → 进入正式游戏
4. 失败 → 弹窗提示"BCI头环未连接，将使用键盘控制"
5. 游戏过程中持续接收注意力 + 陀螺仪 + 焦点坐标数据

### IPC 通信协议

**发送格式**：4 字节大端长度 + UTF-8 JSON 负载

**就绪消息**：

```json
{"type": "ready", "client": "crazy_milk_tea_cup"}
```

**启动陀螺仪算法**：

```json
{
  "msg": "ipc_algorithm_start_test",
  "algorithm_name": "gyroscope",
  "algorithm_args": {
    "left": 0, "top": 0,
    "width": 1280, "height": 720,
    "sensitivityX": 8, "sensitivityY": 8
  }
}
```

**接收数据格式**：

```json
{
  "msg": "ipc_algorithm_test",
  "algorithm_name": "attention",
  "result_args": {"data": 75}
}
```

```json
{
  "msg": "ipc_algorithm_test",
  "algorithm_name": "gyroscope",
  "result_args": {
    "data": {
      "focus_x": 640, "focus_y": 360,
      "gyroscope_x": 9.5, "gyroscope_y": 258.0, "gyroscope_z": 176.0
    }
  }
}
```

### 数据读取机制

- **批量消费**：每帧处理缓冲区中**所有**到达的消息，只保留最新值，消除积压延迟
- **超时断开**：超过 2 秒未收到新数据 → 自动标记连接断开
- **阻塞分离**：socket 设为非阻塞，用 `BlockingIOError` 处理无数据情况
- **实时显示**：注意力数据不使用平滑，直接实时反映变化

所有滤波器位于 `bci/filter.py`。

### 1. 死区滤波（DeadZoneFilter）

消除微小抖动信号，当输入绝对值 < 阈值时输出 0：

```
filter(x) = 0           , 若 |x| < threshold
          = x           , 否则
```

默认阈值 = 5.0。

### 2. 指数平滑（ExponentialSmoothing）

使信号变化更平滑，减少抖动：

```
y_0 = x_0                                            （首次直接取输入）
y_n = α × x_n + (1 - α) × y_{n-1}                   （迭代公式）
```

| α 值       | 响应速度 | 抖动程度 | 适用场景 |
| ----------- | -------- | -------- | -------- |
| 0.1         | 慢       | 非常平滑 | 极静环境 |
| 0.3（默认） | 中等     | 适度平滑 | 正常使用 |
| 0.7         | 快       | 微抖动   | 快速响应 |

### 3. 灵敏度曲线（SensitivityCurve）

非线性映射，小幅度更灵敏，大幅度更稳定：

```
y = sign(x) × base_sensitivity × |x| ^ exponent
```

默认参数：`base_sensitivity = 1.0, exponent = 1.5`

`exponent > 1` 时，小幅信号被压缩更多，需要更大动作才能触发大位移。

### 4. 专注力→速度曲线（AttentionToSpeedCurve）

3 段线性，以校准基线为参考：

```
速度(attn) = ┌ Vmax                            , attn ≤ baseline - 20
             │ Vmax - (Vmax-Vmin) × t          , baseline-20 < attn ≤ baseline+20
             └ Vmin                            , attn > baseline + 20
                          其中 t = (attn - lo) / (hi - lo)
```

默认参数：`speed_min = 2.0, speed_max = 5.0, baseline = 60.0`

专注力越高 → 食材越慢 → 更易接住；专注力越低 → 食材越快。

### 5. 难度自适应（DifficultyAdapter）

根据 30 秒滚动窗口内的平均专注力动态调整难度基线，同时结合**逐杯基线**（每杯独立计算均值作为下杯参考）：

```
samples = [(t, attention), ...]                       # 保留最近 30 秒样本
avg = mean(samples)                                   # 窗口均值
baseline = clamp(avg, DIFFICULTY_BASELINE_MIN, DIFFICULTY_BASELINE_MAX)
```

基线用于秘方触发阈值计算：`secret_threshold = baseline + 10`

**逐杯基线**（`_update_attention_variance` 方差计算的偏移基准）：

```
第1杯基线 = 默认值 40
第N杯基线 = 第N-1杯所有注意力样本的均值
```
