# 疯狂奶茶杯

基于 Pygame 的脑机接口（BCI）奶茶制作游戏。玩家通过键盘或头环控制奶茶杯，接住从天而降的食材，完成一杯杯奶茶。通过科创平台实时获取专注力和头动数据，专注力影响食材速度和收益系数，最终累积营业额提升等级。

## 车道系统

### 六车道配置

画面分为 **6 条垂直车道**，每条宽约 213px（1280÷6）：

```
┌──────┬──────┬─────────────┬──────┬──────┐
│ 车道0 │ 车道1 │  车道2+3合并  │ 车道4 │ 车道5 │
│ 掉落  │ 掉落  │  无分割线    │ 掉落  │ 掉落  │
│ 小料  │ 小料  │  不掉落食材  │ 小料  │ 小料  │
└──────┴──────┴─────────────┴──────┴──────┘
```

- **中间两条车道（车道2、3）**：合并为一个宽区域，无分割线，**不掉落食材**
- **左右四条车道（车道0、1、4、5）**：随机掉落食材
- 每条车道最多同时有 1 个食材（由 `_free_lanes()` 占用检测保证）

### 车道占用检测算法

```
free_lanes = []
for lane in [0, 1, 4, 5]:
    occupied = any(ing.centerx // LANE_WIDTH == lane)
    if not occupied:
        free_lanes.append(lane)
```

### 食材生成位置

随机从空闲车道中选取一条，在该车道宽度范围内随机生成 X 坐标：

```
lane = random.choice(free_lanes)
x = random.randint(lane * LANE_WIDTH, (lane + 1) * LANE_WIDTH - INGREDIENT_SIZE)
```

## 热身阶段

进入正式游戏前，先进入 **3 分钟热身阶段**（`WARMUP_DURATION = 180`），目的是收集玩家的注意力数据，用于正式游戏的个性化难度归一化。

### 热身流程

```
热身介绍(3s) → 热身游戏(3min) → 热身总结(5s) → 正式游戏
```

### 热身玩法

- 与正式游戏相同的接食材玩法，**不计分、不设杯子系统**
- 食材持续掉落，玩家用杯子接住
- 屏幕顶部显示"热身阶段"和倒计时 `MM:SS`

### 热身速度调节（3 秒平滑窗口）

取最近 3 秒注意力滑动平均值，映射到 3 个速度档位：

| 注意力范围（平滑后） | 速度                       | 含义           |
| -------------------- | -------------------------- | -------------- |
| 0 ~ 20               | 冻结                       | 画面和时间停止 |
| 20 ~ 46              | `WARMUP_SPEED_MAX` (6.0) | 快速（不专注） |
| 47 ~ 72              | 中速 (3.75)                | 中等           |
| 73 ~ 100             | `WARMUP_SPEED_MIN` (1.5) | 慢速（高专注） |

速度变化幅度大（1.5 ~ 6.0），**感知明显**，鼓励玩家尽快找到专注状态。

### 低注意力冻结机制

- **冻结触发**：实际注意力值**持续 5 秒 ≤ 15** → 画面和时间冻结，显示"请调整身心状态"
- **解冻恢复**：注意力**持续 5 秒 > 15** → 恢复正常游戏和倒计时
- 冻结期间杯子仍可移动，食材和倒计时暂停

### 热身结束 — 归一化基准计算

热身结束后，取**最后 30 秒**注意力数据，计算个性化归一化范围：

```
上界 = 最后30秒最高注意力（上限 100）
下界 = 最后30秒平均注意力 - 10（下限 0）
范围 = [下界, 上界]

若 上界 - 下界 < 10：向两侧各扩展 5，保证范围宽度至少为 10
若 无有效数据：默认范围 [30, 70]
```

将热身阶段导出的范围映射到 1-100，作为正式游戏的**归一化基准**：

```
归一化值 = clamp((实时注意力 - 下界) / (上界 - 下界) × 99 + 1, 1, 100)
```

## 正式游戏机制

### 一杯制规则

- 一局最大 **12 分钟（720 秒）**，共 **36 杯**，每杯最长 **20 秒**（`CUP_DURATION = 20`）
- **杯结束结算逻辑**（`CupManager.settle_cup()`）：

  1. 无必接食材 → 本杯金额 = 0
  2. 有必接食材 → 本杯金额 = 所有接住食材分值之和
  3. 接到秘方 → 本杯金额 × 2

### 食材分值表

| 食材       | 分值 | 食材         | 分值 | 食材   | 分值 |
| ---------- | ---- | ------------ | ---- | ------ | ---- |
| 珍珠       | 1    | 椰果         | 1    | 牛奶   | 2    |
| 红茶       | 2    | 绿茶         | 2    | 芋圆   | 2    |
| 脆啵啵     | 2    | 芒果         | 3    | 椰奶   | 3    |
| 草莓       | 3    | 芋泥         | 3    | 燕麦奶 | 4    |
| 咖啡       | 4    | 特调稀奶油顶 | 5    | 米酿   | 5    |
| 咸芝士奶盖 | 6    | 茉莉花茶     | 6    |        |      |

### 专注力收益系数（BCI 模式）

防止低专注力下食材速度快反而接到更多、收益更高。高专注力 → 全额收益，低专注力 → 收益打折：

```
系数(norm) = ┌ 1.0                        , norm ≥ 80
             │ 0.5 + 0.5 × (norm-40)/40   , 40 ≤ norm < 80
             └ 0.5                        , norm < 40
```

| 归一化值 | 系数      | 含义             |
| -------- | --------- | ---------------- |
| ≥ 80    | 1.0       | 全额收益         |
| 40 ~ 80  | 0.5 ~ 1.0 | 线性过渡         |
| < 40     | 0.5       | 惩罚（最低保障） |

杯结算时：`最终金额 = 原始金额 × 系数`

### 正式游戏速度调节（归一化线性映射）

正式游戏使用热身阶段导出的归一化范围，将实时注意力动态映射到食材速度：

```
速度 = FORMAL_SPEED_MAX - (归一化值 - 1) / 99 × (FORMAL_SPEED_MAX - FORMAL_SPEED_MIN)
     = 4.5 - (归一化值 - 1) / 99 × 2.5
```

| 归一化值      | 速度                   | 含义 |
| ------------- | ---------------------- | ---- |
| 1（低专注）   | FORMAL_SPEED_MAX (4.5) | 较快 |
| 50            | 3.25                   | 中等 |
| 100（高专注） | FORMAL_SPEED_MIN (2.0) | 较慢 |

速度变化范围 2.0 ~ 4.5（差 2.5），远小于热身阶段的 1.5 ~ 6.0（差 4.5），变化感知减轻。

### 生成间隔动态调节

生成间隔随速度自适应，使食材密度保持一致：

```
速度比 = 当前速度 / 基础速度
调整后间隔 = 基础间隔 × (0.7 + 0.6 × 速度比)
最终间隔 ∈ [0.3, 3.0]
```

### 4 级食材与等级系统

| 等级 | 升级条件    | 可用食材（含前级）              | 必接食材（含前级） |
| ---- | ----------- | ------------------------------- | ------------------ |
| Lv.1 | 默认        | 珍珠、椰果、牛奶、红茶、绿茶    | 牛奶、红茶、绿茶   |
| Lv.2 | 累计 ≥ 80  | +芋圆、脆啵啵                   | +芒果、椰奶        |
| Lv.3 | 累计 ≥ 250 | +草莓、芋泥                     | +燕麦奶、咖啡      |
| Lv.4 | 累计 ≥ 600 | +特调稀奶油顶、米酿、咸芝士奶盖 | +茉莉花茶          |

每级保留所有前级食材，等级越高可选种类越多。

### 必接概率（注意力方差自适应）

以**热身阶段最后 30 秒的平均专注力 μ** 为基线，计算实时注意力与基线的偏离方差：

```
offset_i = 实时注意力_i - μ（热身最后30秒均值）
方差 = Σ(offset_i - offset̄)² / N     （60帧滑动窗口）
```

| 方差     | 模式     | 必接概率 | 含义                       |
| -------- | -------- | -------- | -------------------------- |
| < 50     | 简单模式 | 70%      | 注意力稳定，更多必接食材   |
| 50 ~ 150 | 中等模式 | 50%      | 正常波动                   |
| > 150    | 困难模式 | 30%      | 注意力波动大，必接食材减少 |

### 低专注保护（正式游戏）

- 注意力 **≤ 15 持续 5 秒** → 游戏暂停，半透明黑化遮罩，大字提示"请调整身心状态"
- 注意力 **≥ 10 持续 5 秒** → 恢复游戏
- 遮罩透明度以指数平滑过渡：`α += (target - α) × 0.05`

### 秘方触发机制

**BCI 模式** — 基于持续高专注触发：

```
threshold = min(88.0, 40 + SECRET_RECIPE_OFFSET) = min(88, 45) = 45
若 注意力 > threshold 持续 4 秒 → 触发秘方掉落
```

**非 BCI 模式** — 基于杯序周期性触发：

- 每 `secret_recipe_cup_interval` 杯触发一次（常规模式每杯触发，BCI 模式每 3 杯触发）

秘方素材接到后具有粒子特效，杯结算金额翻倍。

### 防伪迹仲裁（BCI 模式）

正常专注时头部有 1-2 度自然微抖，若检测到异常（刻意咬牙、僵直）则触发惩罚：

- **检测条件**：陀螺仪三轴变化 **均 < 0.5°** 且专注力 **> 80**，持续 **5 秒**
- **惩罚**：屏幕提示"请放松面部肌肉"，画面和时间冻结 **5 秒**，自动恢复
- 恢复后重置陀螺仪基准，防止连续触发

### 最后 5 分钟专注力记录

每局游戏结束时，根据专注力采样和时间推算最近 5 分钟（300 秒）的平均专注力：

```
sps = len(focus_samples) / game_duration           # 每秒采样数
n = max(1, int(sps × 300))                          # 最后 300 秒的样本数
last_5min_avg = mean(focus_samples[-n:])             # 尾部样本均值
```

该值存入历史记录，作为趋势曲线的数据点。

## 记忆模式

### 游戏流程

```
[规则弹窗 3.5s] ← 整个游戏只出现一次
     ↓
┌── 循环 ───────────────────────────────────┐
│ [记忆阶段 2s]  食材图片并排显示 + 配方名    │
│ [接食材 15s]   5车道随机掉落，按序接食材    │
│ [结果显示 1.5s] 成功/失败判定              │
│ [休息 2s]      显示等级/进度统计           │
└──────────────────────────────────────────┘
```

### 核心规则

1. **记忆配方**：屏幕中央显示 2~5 种食材图片和配方名称，停留 2 秒
2. **按序接住**：食材消失后，所有食材随机在 5 个车道掉落，**必须按配方顺序依次接住**
3. **严格顺序**：接错顺序即本轮失败。15 秒内未接完所有配方食材 → 失败
4. **穿插投放**：配方食材按 `_recipe_ratio = 3` 的比例穿插在随机干扰食材中出现（每 3 个中有 1 个是目标食材）

### 难度进阶

| 等级 | 配方食材数 | 晋级/降级条件         |
| ---- | ---------- | --------------------- |
| Lv1  | 2          | 连续成功 3 次 → 升级 |
| Lv2  | 3          | 连续成功 3 次 → 升级 |
| Lv3  | 4          | 连续成功 3 次 → 升级 |
| Lv4  | 5（上限）  | 保持在最高级          |

- 失败 → 连续成功清零
- 累计失败 2 次 → 降一级 + 失败计数清零

### 车道生成算法

记忆模式使用**独立的 5 车道系统**（不同于主游戏的 6 车道），每条车道宽 256px：

```
lane = _free_lane()       # 随机选取空闲车道
center = lane × 256 + 128  # 车道中心
x = random(center-30, center+30)  # 微调 x 坐标
```

车道空闲判断：该车道内食材数 < `_max_per_lane`（当前为 1）且无食材 `y < _lane_spacing`（400px）。

### 关键参数

| 参数                  | 值           | 说明                    |
| --------------------- | ------------ | ----------------------- |
| `_spawn_interval`   | 0.8s         | 生成间隔                |
| `_max_per_lane`     | 1            | 每车道最大食材数        |
| `_lane_spacing`     | 400          | 同车道垂直阻断间距      |
| `_ingredient_speed` | 5.5 px/frame | 食材下落速度            |
| `_recipe_ratio`     | 3            | 目标食材:干扰食材 = 1:2 |
| `_memorize_time`    | 2s           | 记忆阶段时长            |
| `_drop_window`      | 15s          | 接食材阶段最大时长      |

## 脑机接口（BCI）系统

### 架构概览

```
[科创平台] ←→ [TCP Socket 127.0.0.1:8000] ←→ [BCIDataReader] ←→ [GameSession]
                              │
                   IPC 二进制协议
               (4字节大端长度 + JSON负载)
```

### 连接流程

1. 主菜单选择常规模式并点"游戏开始"
2. 尝试 TCP 连接科创平台（默认 `127.0.0.1:8000`）
3. 成功 → 发送就绪消息 → 启动陀螺仪焦点算法 → 进入热身阶段
4. 失败 → 弹窗提示"BCI头环未连接，将使用键盘控制"
5. 游戏过程中持续接收注意力 + 陀螺仪 + 焦点坐标数据

### IPC 通信协议

**发送格式**：4 字节大端长度 + UTF-8 JSON 负载

**就绪消息**：

```json
{"type": "ready", "client": "crazy_milk_tea_cup"}
```

**启动陀螺仪算法**：

```json
{
  "msg": "ipc_algorithm_start_test",
  "algorithm_name": "gyroscope",
  "algorithm_args": {
    "left": 0, "top": 0,
    "width": 1280, "height": 720,
    "sensitivityX": 8, "sensitivityY": 8
  }
}
```

**接收数据格式**：

```json
{
  "msg": "ipc_algorithm_test",
  "algorithm_name": "attention",
  "result_args": {"data": 75}
}
```

```json
{
  "msg": "ipc_algorithm_test",
  "algorithm_name": "gyroscope",
  "result_args": {
    "data": {
      "focus_x": 640, "focus_y": 360,
      "gyroscope_x": 9.5, "gyroscope_y": 258.0, "gyroscope_z": 176.0
    }
  }
}
```

### 数据读取机制

- **批量消费**：每帧处理缓冲区中**所有**到达的消息，只保留最新值，消除积压延迟
- **超时断开**：超过 2 秒未收到新数据 → 自动标记连接断开
- **阻塞分离**：socket 设为非阻塞，用 `BlockingIOError` 处理无数据情况

### 3 秒滚动注意力平均

```python
# 保留最近 3 秒的 (时间戳, 注意力值) 记录
_attention_history = [(t1, v1), (t2, v2), ...]
# 剔除过期记录
valid = [(t, v) for t, v in history if now - t <= 3.0]
# 取均值
rolling_avg = mean(v for _, v in valid)
```

用于热身阶段速度计算和 HUD 显示。

## 信号滤波与映射算法

所有滤波器位于 `bci/filter.py`。

### 1. 死区滤波（DeadZoneFilter）

消除微小抖动信号，当输入绝对值 < 阈值时输出 0：

```
filter(x) = 0           , 若 |x| < threshold
          = x           , 否则
```

默认阈值 = 5.0。

### 2. 指数平滑（ExponentialSmoothing）

使信号变化更平滑，减少抖动：

```
y_0 = x_0                                            （首次直接取输入）
y_n = α × x_n + (1 - α) × y_{n-1}                   （迭代公式）
```

| α 值       | 响应速度 | 抖动程度 | 适用场景 |
| ----------- | -------- | -------- | -------- |
| 0.1         | 慢       | 非常平滑 | 极静环境 |
| 0.3（默认） | 中等     | 适度平滑 | 正常使用 |
| 0.7         | 快       | 微抖动   | 快速响应 |

### 3. 灵敏度曲线（SensitivityCurve）

非线性映射，小幅度更灵敏，大幅度更稳定：

```
y = sign(x) × base_sensitivity × |x| ^ exponent
```

默认参数：`base_sensitivity = 1.0, exponent = 1.5`

`exponent > 1` 时，小幅信号被压缩更多，需要更大动作才能触发大位移。

### 4. 专注力→速度曲线（AttentionToSpeedCurve）

3 段线性，以校准基线为参考：

```
速度(attn) = ┌ Vmax                            , attn ≤ baseline - 20
             │ Vmax - (Vmax-Vmin) × t          , baseline-20 < attn ≤ baseline+20
             └ Vmin                            , attn > baseline + 20
                          其中 t = (attn - lo) / (hi - lo)
```

默认参数：`speed_min = 2.0, speed_max = 5.0, baseline = 60.0`

专注力越高 → 食材越慢 → 更易接住；专注力越低 → 食材越快。

### 5. 专注力评分映射（AttentionMappingCurve）

用于创意模式，将专注力转换为配方评分加成倍率：

```
倍率(attn) = ┌ 0.5 + attn/30 × 0.3                    , attn < 30
             │ 0.8 + (attn-30)/40 × 0.2                , 30 ≤ attn < 70
             └ 1.0 + ((attn-70)/30)^0.7 × 0.5          , 70 ≤ attn ≤ 100
```

| 专注力范围 | 倍率范围  | 等级     |
| ---------- | --------- | -------- |
| 0 ~ 30     | 0.5 ~ 0.8 | 分心状态 |
| 30 ~ 70    | 0.8 ~ 1.0 | 平稳专注 |
| 70 ~ 100   | 1.0 ~ 1.5 | 高度专注 |

### 6. 难度自适应（DifficultyAdapter）

根据 30 秒滚动窗口内的平均专注力动态调整难度基线：

```
samples = [(t, attention), ...]                       # 保留最近 30 秒样本
avg = mean(samples)                                   # 窗口均值
baseline = clamp(avg, DIFFICULTY_BASELINE_MIN, DIFFICULTY_BASELINE_MAX)
```

基线用于秘方触发阈值计算：`secret_threshold = min(88.0, baseline + SECRET_RECIPE_OFFSET)`

## 账户与存档系统

- 首次启动需创建账号密码（存于 `accounts.json`）
- 每个账号独立存档（`profiles/用户名.json`）
- JSON 存档格式：

```json
{
  "level": 2,
  "cumulative_revenue": 156,
  "total_games": 5,
  "games_history": [
    {
      "mode": "bci",
      "revenue": 42,
      "cups": 36,
      "secrets": 2,
      "avg_attention": 65.3,
      "last_5min_avg_attention": 72.1,
      "duration": 645.2,
      "date": "2026-06-03 14:30",
      "focus_samples": [50, 52, 55, ...]
    }
  ]
}
```

### 等级阈值

| 等级 | 累计营业额门槛 |
| ---- | -------------- |
| Lv.1 | 0              |
| Lv.2 | 80             |
| Lv.3 | 250            |
| Lv.4 | 600            |

### 存档操作

- `add_game_result()` — 添加对局记录，自动累加营收、检查升级
- `remove_game(idx)` — 删除单条记录，自动重算营收和等级
- `clear_history()` — 清空所有记录，重置统计
- `save()` — 持久化到 JSON，保留最近 20 条记录
- `_recalculate()` — 重算 total_games、cumulative_revenue、level
- 只保留最近 20 条历史记录

## 历史记录系统

主菜单点击徽章进入历史记录界面。

### 界面布局（左右分栏）

```
┌─────────────────────┬──────────────────────────────┐
│  历史记录            │  [一键清除]                    │
│ ─────────────────── │ ─────────────────────────────│
│                     │   时间-专注力趋势曲线           │
│  记录列表（滚动）     │                              │
│  + 左键查看详情       │   X 轴: 日期 (MM-DD)          │
│  + 右键删除单条       │   Y 轴: 最后5分钟平均专注力    │
│                     │   仅显示"常规"模式对局          │
│                     │                              │
│                     │   ┌──────────────────────┐   │
│                     │   │ 文字框（预留）        │   │
│                     │   └──────────────────────┘   │
└─────────────────────┴──────────────────────────────┘
```

### 操作功能

- **左键**点击列表项 → 弹出详情窗口（含游戏数据 + 专注力波形图）
- **右键**点击列表项 → 弹出确认框 → 删除该条记录
- **一键清除**按钮（标题右侧，小号按钮）→ 弹出确认框 → 清空全部历史
- **滚轮**翻页浏览
- 删除后自动重算等级并即时保存

### 专注力波形图

详情窗口中绘制单局专注力随时间变化的波形图：

- 采样降频至最多 150 个点（分桶取均值）
- X 轴：时间（0s ~ 游戏时长）
- Y 轴：专注力（0 ~ 100）
- 网格线每 25 单位
