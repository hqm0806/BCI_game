# 疯狂奶茶杯

基于 Pygame 的脑机接口（BCI）奶茶制作游戏。玩家通过键盘或头环控制奶茶杯，接住从天而降的食材，完成一杯杯奶茶。通过科创平台实时获取专注力和头动数据，专注力影响食材速度和必接概率，最终累积营业额提升等级。

## 记忆模式

### 游戏流程

```
[游戏介绍弹窗 4s]  ← 整个游戏只出现一次
     ↓
┌── 循环 ──────────────────────────────┐
│ [记忆阶段 3s]  食材图片并排显示 + 配方名 │
│ [接食材 15s]   5车道随机掉落，按序接食材 │
│ [结果显示 1.5s] 成功制作 / 制作失败     │
│ [休息 2s]      显示等级/进度统计       │
└─────────────────────────────────────┘
```

### 规则

1. **记忆配方**：屏幕中央显示 2~5 种食材图片和配方名称，停留 3 秒
2. **按序接住**：食材消失后，所有食材随机在 5 个车道掉落，必须按配方顺序依次接住
3. **严格顺序**：接错顺序即本轮失败。15 秒内未接完所有配方食材 → 失败
4. **所有食材统一管理**：配方食材按比例穿插在随机食材中出现，5 车道每车道最多 2 个食材，同车道间距 ≥ 400px

### 难度进阶

| 等级 | 配方食材数 | 晋级条件 |
|------|-----------|---------|
| Lv1 | 2 | 连续成功 3 次 → 升级 |
| Lv2 | 3 | 连续成功 3 次 → 升级 |
| Lv3 | 4 | 连续成功 3 次 → 升级 |
| Lv4 | 5（上限） | 保持在最高级 |

- 失败 → 连续成功清零
- 累计失败 2 次 → 降一级 + 失败计数清零
- HUD 实时显示：难度等级、配方食材数、得分、连续成功/失败统计

## 热身阶段

进入正式游戏前，先进入 **3 分钟热身阶段**，目的是收集玩家的注意力数据，用于正式游戏的个性化难度归一化。

### 热身玩法

- 与正式游戏相同的接小料玩法，**不计分、不设杯子系统**
- 食材持续掉落，玩家用杯子接住
- 屏幕顶部显示"热身阶段"和倒计时 `MM:SS`

### 速度调节（3 秒平滑窗口）

取最近 3 秒注意力滑动平均值，映射到 3 个速度档位：

| 注意力范围（平滑后） | 速度 | 含义 |
|---|---|---|
| 0 - 20 | 冻结（画面 + 时间停止） | — |
| 20 - 46 | `WARMUP_SPEED_MAX` (6.0) | 快速（不专注） |
| 47 - 72 | 中速 (3.75) | 中等 |
| 73 - 100 | `WARMUP_SPEED_MIN` (1.5) | 慢速（高专注） |

速度变化幅度大（1.5 ~ 6.0），**感知明显**，鼓励玩家尽快找到专注状态。

### 低注意力冻结机制（与正式游戏一致）

- **冻结触发**：实际注意力值**持续 5 秒 ≤ 15** → 画面和时间冻结，显示"请调整身心状态"
- **解冻恢复**：注意力**持续 5 秒 > 15** → 恢复正常游戏和倒计时
- 冻结期间杯子仍可移动，食材和倒计时暂停

### 热身结束 — 归一化

热身结束后，取**最后 30 秒**注意力数据：

```
下界 = max(平均注意力 - 10, 0)
上界 = 最高注意力（上限 100）
范围 = [下界, 上界]（若宽度不足 10，自动扩展到 10）
```

将热身阶段导出的范围映射到 1-100，作为正式游戏的**归一化基准**：

```
归一化值 = clamp((实时注意力 - 下界) / (上界 - 下界) × 99 + 1, 1, 100)
```

## 正式游戏机制

### 五等分车道系统

- 画面分为 **5 条垂直车道**，每条宽 256px（1280÷5）
- **中间车道（车道3）**：只显示专注力值，不掉落小料
- **左右四条车道（车道1/2/4/5）**：随机掉落小料，**每条车道最多同时只有 1 个小料**
- 车道之间有半透明灰色分割线

### 一杯制规则

- 一局 **12 分钟（720 秒）**，共 **36 杯**，每杯最长 **20 秒**
- 杯结束时：无必接食材 → 金额 0；有必接 → 累加所有接住食材值
- 秘方翻倍：接到秘方 → 本杯结算金额 **×2**
- **专注力收益系数**（BCI 模式）：结算金额 × 系数 a
    - 归一化值 ≥ 80 → a = **1.0**（全额）
    - 归一化值 40~80 → a = **0.5~1.0**（线性过渡）
    - 归一化值 < 40 → a = **0.5**（惩罚，防止高速食材带来超额收益）
- 右上角 HUD 显示已接杯数

### 速度调节（归一化线性映射）

正式游戏使用热身阶段导出的归一化范围，将实时注意力动态映射到速度：

```
速度 = FORMAL_SPEED_MAX - (归一化值 - 1) / 99 × (FORMAL_SPEED_MAX - FORMAL_SPEED_MIN)
      = 4.5 - (归一化值 - 1) / 99 × 2.5
```

| 归一化值 | 速度 | 含义 |
|---|---|---|
| 1（低专注） | `FORMAL_SPEED_MAX` (4.5) | 较快 |
| 50 | 3.25 | 中等 |
| 100（高专注） | `FORMAL_SPEED_MIN` (2.0) | 较慢 |

速度变化范围 2.0 ~ 4.5（差 2.5），**远小于热身阶段的 1.5 ~ 6.0（差 4.5）**，变化感知明显减轻。

### 4 级食材系统

| 等级 | 升级条件 | 选接食材 | 必接食材 |
|------|---------|---------|---------|
| Lv.1 | 默认 | 珍珠(1) 椰果(1) | 牛奶(2) 红茶(2) 绿茶(2) |
| Lv.2 | 累计 ≥ 80 | +芋圆(2) 脆啵啵(2) | +芒果(3) 椰奶(3) |
| Lv.3 | 累计 ≥ 250 | +草莓(3) 芋泥(3) | +燕麦奶(4) 咖啡(4) |
| Lv.4 | 累计 ≥ 600 | +稀奶油顶(5) 米酿(5) 咸芝士奶盖(6) | +茉莉花茶(6) |

每级保留所有前级食材，等级越高可选种类越多。

### 必接概率（注意力方差调节）

以**热身阶段最后 30 秒的平均专注力 μ** 为基线，计算实时注意力与基线的偏离方差：

```
offset = 实时注意力 - μ（热身最后30s均值）
方差 = Σ(offsetᵢ - offset̄)² / N   （60帧滑动窗口）
```

| 方差 | 模式 | 必接概率 | 含义 |
|------|------|---------|------|
| < 50 | 简单 | 70% | 注意力稳定，更多必接食材 |
| 50-150 | 中等 | 50% | 正常波动 |
| > 150 | 困难 | 30% | 注意力波动大，必接食材减少 |

### 低专注保护（正式游戏）

- 注意力 **≤ 15 持续 5 秒** → 游戏暂停，屏幕黑化，大字提示"请调整身心状态"
- 注意力 **≥ 10 持续 5 秒** → 恢复游戏

### 防伪迹仲裁（正式游戏 BCI 模式）

正常专注时头部有 1-2 度自然微抖，若检测到异常则触发惩罚：

- **检测条件**：头部（陀螺仪三轴）**持续 5 秒静止**（变化 < 0.5°）且专注力 **> 80**
- **惩罚**：屏幕提示"请放松面部肌肉"，画面和时间冻结 **5 秒**，自动恢复
- 恢复后重置陀螺仪基准，防止连续触发

## 脑机接口

### 连接流程

1. 主菜单选择常规模式并点"游戏开始"
2. 尝试连接 BCI，失败弹窗提示后降级为键盘控制
3. 连接成功进入热身阶段（3 分钟），开始接收注意力数据
4. 热身阶段使用头环控制奶茶杯 + 专注力影响食材速度
5. 热身结束后自动进入正式游戏

### 协议

基于 IPC 通信协议，通过 TCP Socket 连接科创平台（默认 `127.0.0.1:8000`）：

```
连接 → 发送就绪消息
     → 发送 ipc_algorithm_start_test (gyroscope)
     ← 接收 ipc_algorithm_test (attention / gyroscope)
     
gyroscope 数据格式:
  {"msg":"ipc_algorithm_test", "algorithm_name":"gyroscope",
   "result_args":{"data":{"focus_x":640, "focus_y":360, 
    "gyroscope_x":9.5, "gyroscope_y":258.0, "gyroscope_z":176.0}}}
```

杯子直接取 `focus_x` 作为屏幕位置，不额外平滑（平台已处理）。

### 配置文件

`bci_config.json`：
```json
{"server_ip": "127.0.0.1", "server_port": 8000}
```
