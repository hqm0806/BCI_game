## 二、总体时间线

```
[主菜单] → [入场动画 SplashScreen + BGM 晨光木盒]
    ↓ (0s)
[原萃阶段弹窗 2.5s] — 画面冻结、计时暂停
    ↓ 弹窗消失
[热身阶段 3min] — O键可跳过
    ↓ 计算基线/归一化
[过渡弹窗 2s] — 显示基线、归一化范围，食材冻结
    ↓
[特调阶段 7min] — O键可跳过
    ↓
[过渡弹窗 2s] — 显示累计杯数、收益，食材冻结
    ↓
[忆调阶段 5min] — 限时到达后本轮结束退出
    ↓
[返回主菜单]
```

### 阶段流转结构

| phase 值 | 含义 | 时长 | 手动跳过 |
|---|---|---|---|
| `warmup` | 热身（原萃驱动） | 3min | O 键 |
| `transition` | warmup→formal 过渡 | 2s | — |
| `formal` | 特调（归一化调度） | 7min | O 键 |
| `transition_memory` | formal→memory 过渡 | 2s | — |
| `memory` | 记忆配方训练 | 5min | — |

## 五、热身阶段（原萃驱动, 3min）

### 流程

```
弹窗(2.5s, 画面冻结) → 计时开始 → 3min 原萃游戏 → 计算基线 → transition(2s)
                     ↑ O键跳过
```

### 算法

| 算法 | 公式 | 参数 |
|---|---|---|
| **注意力→速度** | `speed = WARMUP_SPEED_MAX - (attn/100) * (MAX-MIN)` | `1.5 ~ 6.0 px/frame` |
| **生成间隔** | `adjusted = spawn_interval * (0.7 + 0.6 * speed_ratio)` | `[0.3, 3.0]s` |
| **秘方阈值** | `threshold = _cup_baseline + 10`（逐杯更新 `_cup_baseline`） | 持续 8s 触发 |
| **收益系数** | `get_attention_coefficient(attn)` — 原始专注力值直接计算 | 3 段分段函数 |
| **杯结算** | `CupManager.settle_cup()` → 秘方翻倍 ×2 → 收益系数 → `add_cup_money` | 仅必接成功杯 |

### 食材生成管线（20/80 比例）

```
每帧 _update_game_objects:
  should_spawn()?
    ├─ 20%: tier_required 中随机 → is_required=True → set_particle_group (100%必带粒子)
    └─ 80%: available-required 中随机 → "冰块" 固定入选接池 → is_required=False
```

### 归一化基准计算 (`_finish_warmup`)

```
baseline = mean(all_warmup_samples)           # 全场均值
norm_max = max(最后30s样本)
norm_min = min(最后30s样本)
若 max - min < 5: 各扩展 10
锁定: normalization_lower = norm_min, normalization_upper = norm_max
```

## 六、特调阶段（归一化调度, 7min）

### 算法（全部锁定热身基线）

| 算法 | 公式 | 锁定值 |
|---|---|---|
| **注意力→归一化** | `norm = (attn - lower) / (upper - lower) * 99 + 1` | `lower, upper ← 热身` |
| **注意力→速度** | `speed = FORMAL_SPEED_MAX - (norm-1)/99 * (MAX-MIN)` | `2.0 ~ 4.5 px/frame` |
| **生成间隔** | `adjusted = spawn_interval * (0.7 + 0.6 * speed_ratio)` | 同热身 |
| **秘方阈值** | `threshold = warmup_baseline + 10` | **固定**，不逐杯更新 |
| **收益系数** | `get_attention_coefficient(norm)` — 归一化后计算 | 3 段分段函数 |
| **杯结算** | 同上（`_check_cup_end` 通用） | — |

### 食材生成管线

与热身阶段完全相同（20/80 + 冰块入选接池）。

### 收益系数函数

```
coeff(norm) = ┌ 1.0                        , norm ≥ 80
              │ 0.5 + 0.5×(norm-40)/40    , 40 ≤ norm < 80
              └ 0.5                        , norm < 40
```

## 七、忆调阶段（记忆训练, 5min）

### 子状态流转

```
rules(3.5s) → memorize(2s) → playing(配方掉落) → result(1.5s) → rest(2s) → memorize(2s) → ...
                                                                          ↑ 下一轮
```

5 分钟限时到期后当前轮结束，自动退出。

### 配方系统

- **配方库**：`data/memory_recipes.MEMORY_RECIPES[level]`，35 个预定义配方
- **等级**：2~5 个食材，初始 Lv.2
- **晋级/降级**：3 连成功 → 升 1 级；2 连失败 → 降 1 级

### 生成序列 (`_generate_spawn_sequence`)

```
recipe_seq = list(recipe) * 2        # 配方食材各 2 次: [A,B,A,B]
total_dist = n * 3                   # 干扰食材 3n 个
→ 均匀交错插入，保持配方顺序
→ 总长度 = 2n + 3n = 5n
```

| n | 配方食材 | 干扰 | 总数 | 生成时间 (×1.2s) |
|---|---|---|---|---|
| 2 | 4 (A,B,A,B) | 6 | 10 | 12s |
| 3 | 6 | 9 | 15 | 18s |
| 4 | 8 | 12 | 20 | 24s |
| 5 | 10 | 15 | 25 | 30s |

### Spawn 逻辑 (`_update_memory_game` playing)

```
每 1.2s:
  sequence[spawn_index] → 获取 ing_type
  _memory_free_outlets() → 检查 4 出料口占用
  有空出口 → Ingredient(outlet_index) → add to group → index+1
  无空出口 → 延迟 0.2s 重试
  index >= len → _memory_all_spawned = True
```

### 碰撞检测 (`_check_memory_collisions`)

- `hit.type == recipe[catch_index]` → catch_index += 1
- `hit.type != recipe[catch_index]` → fail_reason = "wrong_order"
- `catch_index >= len(recipe)` 时：其他食材不判断（无害通过）

### 成功/失败判定

| 条件 | 结果 | 显示 |
|---|---|---|
| `catch_index >= len(recipe)`，无 wrong_order | **正确调配** | 绿色 + 配方名 |
| 接错食材 | **错误调配** | 红色 |
| 全部食材掉完，`catch_index < len(recipe)` | **未完成** | 橙色 |

### 结算（result）

- 成功：`add_cup_money(recipe_len * 10)` → 信息栏杯数+1，收益递增
- 失败：仅累计连胜/连败统计，不影响杯数和收益
- 5 分钟限时到期 → `_memory_session_ending = True` → 本轮结束后 `running = False`

### 忆调子阶段 HUD

| 子阶段 | 显示内容 |
|---|---|
| rules | 深色遮罩 + 白字: "忆调阶段 — 记忆配方，按序接食材", "1. 记住下方食材和名称", "2. 按配方顺序依次接住" |
| memorize | 深色遮罩 + 配方食材图片 + 配方名 + 记忆计时条 |
| playing | 杯子可移动，食材从出料口下落 |
| result | 深色遮罩 + 正确调配/错误调配/未完成 + 配方名 |
| rest | "下一杯即将开始... Ns" (黑色文字, y=180) |

```
[HybridBCI 平台]
    │ TCP :8000
    ▼
BCIDataReader.read_with_timeout()
    │ (attention, focus_x, focus_y, gyro_x, gyro_y, gyro_z)
    ▼

    │ self.attention, self.platform_focus_x/y, self.raw_gyro_x/y/z
    ├─ self._warmup_samples.append((now, attention))    # 热身→基线
    └─ self._cup_attn_samples.append(attention)          # 杯级样本
         ▼
    speed 计算 → ingredient/spawn 管控
         ▼
    碰撞 → ScoreManager + CupManager (warmup/formal)
          → 记忆碰撞验证 (memory)
         ▼
    结算 → add_cup_money → 信息栏更新
         ▼
    阶段计时 → phase 切换 → 结束
```
